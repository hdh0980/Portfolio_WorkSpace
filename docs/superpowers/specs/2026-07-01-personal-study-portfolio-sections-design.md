# 개인 학습 포트폴리오 섹션 개편 설계

## 목적

`studies/전공학습/01_전기전자_핵심기초/`의 Markdown 문서를 채용 홍보물이나
문제집이 아니라, 작성자가 나중에 다시 읽을 수 있는 개인 기술 포트폴리오로
정리한다. 각 문서는 무엇을 이해했는지, 어떤 계산이나 자료로 확인했는지,
다음에 무엇을 확장할지를 빠르게 되짚을 수 있어야 한다.

## 대상

- 회로이론, 전자기학, 전자회로, 논리회로, 신호와 시스템의 단원 문서
- 위 문서를 안내하는 과목별 및 상위 `README.md`
- 현재 작업 트리에 남아 있는 사용자 수정은 보존한다.

## 개편 원칙

1. 단원별 `실무 연결과 주의점`과 이미 제거된 `자가 점검`의 자리를
   `학습·검증 기록`으로 통합한다.
2. `학습·검증 기록`은 다음 세 요소로 구성한다.
   - **핵심 정리:** 해당 단원에서 정리한 개념과 판단 기준
   - **확인 근거:** 본문 계산 예제, 비교표, 시뮬레이션 또는 공개 참고자료 중
     실제 문서 안에서 확인할 수 있는 근거
   - **다음 탐구:** 현재 범위를 넘어 이어서 살펴볼 구체적인 주제
3. 모든 문서에 같은 상투문을 반복하지 않고 단원 내용에 맞게 작성한다.
4. 수행하지 않은 실험, 시뮬레이션, 프로젝트 경험은 새로 주장하지 않는다.
5. 기존 본문, 수식, 계산 예제, 참고자료는 대상 섹션과 직접 충돌하지 않는 한
   유지한다.
6. 번호가 붙은 본문 구조와 독립되도록 새 섹션 제목은 번호 없이 사용한다.

## 상위 문서 반영

과목 및 전공학습 안내 문서에서 `자가 점검`, `실무 주의점`처럼 문제집 또는
업무 지침으로 읽히는 표현을 제거한다. 대신 `핵심 정리`, `계산·검증 기록`,
`후속 탐구`가 이 자료의 공통 구성임을 안내한다.

## 검증 기준

- 대상 경로에서 `실무 연결과 주의점`, `자가 점검`, `자가점검` 문구가 남지 않는다.
- 모든 단원 문서에 `학습·검증 기록`이 정확히 한 번 존재한다.
- 각 기록에는 핵심 정리, 확인 근거, 다음 탐구가 모두 있다.
- Markdown 제목 구조, 로컬 링크, UTF-8 인코딩이 유지된다.
- Git diff에서 기존 사용자 수정이 되살아나거나 무관한 파일이 변경되지 않는다.

## 제외 범위

- 새로운 이미지, MATLAB 코드 또는 실험 결과 생성
- 기술 내용 전체의 재작성
- 원격 저장소 push
- 제어시스템 설계 영역과 저장소 루트의 포트폴리오 자산 변경
