# UL 1741·UPS 포트폴리오 자산 개편 설계

## 목적

GitHub의 `assets/portfolio/` 폴더를 열었을 때 파일 목록과 파형만 보이는 문제를
해결한다. UL 1741 자료는 시험 목적과 케이스를 먼저 이해할 수 있게 정리하고,
실제 파형 영역은 공개용 이미지에서 식별할 수 없도록 처리한다. UPS 자료는 UL
1741과 분리하여 각 파형이 보여 주는 검증 관점을 설명한다.

## 공개 진입점

`assets/portfolio/README.md`를 새로 만든다. GitHub가 폴더 하단에 자동으로
렌더링하는 이 문서를 다음 순서로 구성한다.

1. 이 폴더의 목적과 공개 범위
2. UL 1741 및 UL 1741 SA의 간단한 정의
3. UL 1741 시험 케이스와 공개용 이미지 매칭
4. UPS 검증 파형
5. 해석 범위와 주의사항
6. UL Solutions·IEEE 공식 참고자료

저장소 루트 `README.md`의 파일명 나열은 상세 문서 링크 중심으로 바꾼다.

## 저장소 루트 README 개편

루트 README는 자산 파일 목록이 아니라 저장소 전체를 탐색하는 시작점으로
재작성한다. 실제 하위 폴더와 작성 현황을 기준으로 다음 순서를 사용한다.

1. 저장소 목적과 Notion 포트폴리오 링크
2. 검증 자산·전공학습·제어시스템 바로가기
3. 실제 폴더 트리와 폴더별 역할
4. UL 1741·UPS 검증 자료 요약
5. 전기전자 핵심기초 46개 단원과 제어시스템 7개 노트·MATLAB 프로젝트
6. 03~10 학습 영역의 계획 상태
7. 공개 원칙과 유지관리 기준

`assets/portfolio/`, `studies/전공학습/`, `docs/superpowers/`를 모두 구조도에
표시하되 `docs/superpowers/`는 작업 설계·계획 기록으로만 설명한다. 아직
README나 본문이 없는 03~10 학습 영역은 완료 콘텐츠처럼 소개하지 않고
`계획`으로 구분한다. 실제로 존재하지 않는 프로필·HILS 파일명과 중복된 PNG
파일 목록은 제거하고, 본문은 한국어 중심으로 통일한다.

## UL 1741 자료의 해석 범위

- UL 1741은 분산에너지자원용 인버터·컨버터·제어기·계통연계 장비를 다루는
  표준이라는 범위만 설명한다.
- 현재 이미지가 `UL 1741 SA` 및 IEEE 1547-2003 시기의 시험 화면을 포함하므로,
  최신 UL 1741 전체 인증 시험이나 현행 IEEE 1547 적합성 보고서라고 표현하지
  않는다.
- 이미지에 표시된 시간·전류·전력 값은 해당 캡처의 시험 설정 또는 결과로
  설명하며, 모든 제품에 공통인 현재 요구값으로 일반화하지 않는다.
- 이 포트폴리오는 시험 수행 범위와 해석 역량을 보여 주는 요약 자료이며,
  공인 인증서나 전체 시험 성적서가 아님을 명시한다.

## 공개용 이미지 처리

기존 다섯 파일명은 외부 링크 안정성을 위해 유지하되, 최신 Git 상태의 파일은
파형 영역을 강하게 모자이크 처리한 공개용 버전으로 교체한다. 처리 전 원본은
Git 저장소 밖의 `D:\이력서\portfolio_waveform_originals_private\`에 복사한다.
기존 Git 이력에는 과거 원본이 남을 수 있으며, 이 작업은 이력 정리까지 포함하지
않는다.

### 파일별 보존·비공개 영역

| 파일 | 보존할 시험 케이스 영역 | 모자이크 처리 영역 |
|---|---|---|
| `ul1741_ess_power_control.png` | 제목, 개발 내용, 채널 범례 | 충전·대기·방전 파형 전체 |
| `ul1741_ramp_rate.png` | 좌측 Ramp Rate 개념도와 4개 설정값 | 우측 네 개 측정 파형 |
| `ul1741_voltage_ride_through.png` | 좌측 VRT 영역도와 HV/LV 케이스 목록 | 우측 측정 파형 전체 |
| `ul1741_frequency_ride_through.png` | 좌측 FRT 영역도와 HF/LF 케이스 목록 | 우측 측정 파형 전체 |
| `ul1741_anti_islanding.png` | 상단 RLC 시험 회로와 4단계 절차 | 하단 측정 파형 전체 |

모자이크 경계는 파형의 진폭, 위상, 응답시간, 축 눈금과 계측값을 읽을 수 없도록
충분히 넓게 잡는다. 공개용 편집 과정에서 출력 크기가 정규화될 수 있으므로
보존 영역의 문자·도식이 판독 가능하고 의미가 바뀌지 않았는지를 눈으로
재검토한다.

## UL 1741 시험 케이스 설명

README는 이미지별로 다음 네 항목을 제공한다.

- **시험 목적:** 무엇을 검증하는가
- **시험 입력·상태:** 어떤 계통 조건이나 운전 상태를 인가하는가
- **확인 항목:** 유지 운전, 출력 제한, 복귀, 정지 등 무엇을 판정하는가
- **이미지 매칭:** 공개 이미지에서 어느 도식·번호를 보면 되는가

다섯 케이스는 다음과 같이 연결한다.

1. ESS Power Control — 충전·대기·방전 상태 전환과 완만한 지령 변화
2. Ramp Rate — 정상 기동 및 소프트스타트의 출력 변화율 설정
3. Voltage Ride-Through — HV2/HV1/NN/LV1/LV2/LV3 전압 영역별 유지·정지
4. Frequency Ride-Through — HF2/HF1/NN/LF1/LF2 주파수 영역별 유지·정지
5. Anti-Islanding — 계통 개방 후 단독운전 검출과 출력 정지

## UPS 섹션

UPS는 별도 `UPS 검증 파형` 섹션으로 분리하고 원본 세 이미지를 그대로 사용한다.

| 파일 | 설명할 검증 관점 |
|---|---|
| `ups_parallel_waveform.png` | 병렬운전 중 출력 전압과 전류 채널의 정상상태 및 부하분담 |
| `ups_load_transient.png` | 부하 단계 변화 전후의 전압 유지와 전류 응답 |
| `ups_hot_swap.png` | 모듈 투입·분리 구간의 전류 이관과 출력 연속성 |

채널 이름이 이미지에 명시되지 않은 경우 특정 모듈 번호나 상을 단정하지 않고,
공개 화면에서 확인 가능한 전압·전류 변화만 설명한다.

## 검증 기준

- GitHub 폴더에서 `README.md`가 렌더링되고 UL 1741과 UPS가 분리되어 보인다.
- UL 1741 다섯 공개 이미지에서 원래 측정 파형과 수치가 판독되지 않는다.
- 시험 케이스 도식과 목록은 판독 가능하다.
- README의 이미지 링크 8개와 공식 참고 링크가 유효하다.
- UL 1741 시험 설명과 이미지 번호가 서로 일치한다.
- 저장소 루트 README가 상세 자산 문서로 연결된다.
- 저장소 루트 README의 트리와 작성 현황이 실제 하위 폴더·파일 수와 일치한다.
- 완료 영역과 계획 영역이 명확히 구분되고 존재하지 않는 자산명이 남지 않는다.
- 변경된 Markdown은 `git diff --check`를 통과한다.
- 최종 커밋 후 `origin/main`에 push하여 사용자가 제공한 GitHub URL에 반영한다.

- 공인 시험성적서 또는 인증서 형태의 문서 작성
