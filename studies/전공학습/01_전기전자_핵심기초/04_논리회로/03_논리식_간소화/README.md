# 논리식 간소화

## 학습 목표
- SOP(최소항의 합)와 POS(최대항의 곱) 표준형을 정의하고 상호 변환할 수 있다.
- 2~4변수 카르노맵을 작성하고 최소화 표현을 구할 수 있다.
- Don't-care(X) 조건을 카르노맵에서 활용하여 추가 간소화를 달성할 수 있다.
- Quine-McCluskey 방법의 기본 절차와 장점을 설명할 수 있다.
- 게이트 수 최소화가 실무 설계(FPGA LUT 절약, 타이밍 개선)에 미치는 영향을 설명할 수 있다.

---

## 1. 표준형 (Canonical Forms)

### 1.1 최소항(Minterm)과 최대항(Maxterm)

n변수 함수에서 각 변수가 보수/비보수 형태로 한 번씩 나타나는 항.

| 행 | A | B | 최소항 m_i | 최대항 M_i |
|----|---|---|-----------|-----------|
| 0 | 0 | 0 | A'B' = m0 | A+B = M0 |
| 1 | 0 | 1 | A'B = m1 | A+B' = M1 |
| 2 | 1 | 0 | AB' = m2 | A'+B = M2 |
| 3 | 1 | 1 | AB = m3 | A'+B' = M3 |

### 1.2 SOP (Sum of Products — 최소항의 합)

Y = 1이 되는 행의 최소항들을 OR로 묶음.

```
Y = Σm(0, 2, 3)  →  Y = A'B' + AB' + AB
```

### 1.3 POS (Product of Sums — 최대항의 곱)

Y = 0이 되는 행의 최대항들을 AND로 묶음.

```
Y = ΠM(1)  →  Y = (A + B')
```

> SOP와 POS는 동치: m_i와 M_i는 서로 쌍대 관계.

## 2. 카르노맵 (Karnaugh Map)

### 2.1 2변수 카르노맵

```
      B=0  B=1
A=0 | m0 | m1 |
A=1 | m2 | m3 |
```

### 2.2 3변수 카르노맵

```
      BC
AB  | 00 | 01 | 11 | 10 |
 0  | m0 | m1 | m3 | m2 |
 1  | m4 | m5 | m7 | m6 |
```

> **그레이 코드 순서**로 열을 배열 (00→01→11→10) — 인접 셀은 1비트만 다름.

### 2.3 4변수 카르노맵

```
        CD
AB    | 00 | 01 | 11 | 10 |
  00  | m0 | m1 | m3 | m2 |
  01  | m4 | m5 | m7 | m6 |
  11  |m12 |m13 |m15 |m14 |
  10  | m8 | m9 |m11 |m10 |
```

### 2.4 그루핑 규칙

| 그룹 크기 | 없애지는 변수 수 | 결과 항의 리터럴 수 |
|-----------|-----------------|-------------------|
| 1 | 0 | 4 |
| 2 | 1 | 3 |
| 4 | 2 | 2 |
| 8 | 3 | 1 |
| 16 | 4 | 0 (항상 1) |

- 그룹 크기는 **2의 거듭제곱**이어야 함.
- 셀은 재사용 가능 (겹치는 그룹 허용).
- 지도는 **토로이드** 형태 — 좌우·상하 경계가 연결됨.

## 3. 카르노맵 간소화 예제

**예제.** F(A,B,C,D) = Σm(0,1,2,5,8,9,10)을 간소화하라.

```
4변수 카르노맵 작성:
        CD
AB    | 00 | 01 | 11 | 10 |
  00  |  1 |  1 |  0 |  1 |
  01  |  0 |  1 |  0 |  0 |
  11  |  0 |  0 |  0 |  0 |
  10  |  1 |  1 |  0 |  1 |

그루핑:
① {m0,m1,m8,m9} (AB=0x, CD=0x 아님... 재확인)
  m0=0000, m1=0001, m8=1000, m9=1001 → A-don't, B=0, C=0, D-don't
  → 그룹 ①: B'C'

② {m0,m2,m8,m10} → m0=0000,m2=0010,m8=1000,m10=1010
  → B=0, D=0 → 그룹 ②: B'D'

③ {m1,m5} → m1=0001,m5=0101 → A=0, C=0, D=1
  → 그룹 ③: A'C'D

최종 SOP: F = B'C' + B'D' + A'C'D
```

## 4. Don't Care (무관 조건)

입력 조합이 실제로 발생하지 않거나 출력이 중요하지 않은 경우 **X**로 표기.

- 카르노맵에서 **X를 1로도 0으로도 취급 가능** — 더 큰 그룹 형성에 활용.
- 반드시 어떤 그룹에 포함시킬 필요는 없음.

예) BCD 입력(10진수 0~9만 유효) → 1010~1111은 don't care.

```
F = Σm(1,3,5,7,9) + d(10,11,12,13,14,15)
don't care를 1로 활용 → m9와 m11,m13 등 묶어 AB 제거 가능
→ 간소화 결과 크게 달라짐
```

## 5. Quine-McCluskey (QM) 방법 — 개요

카르노맵은 6변수 초과 시 시각화 어려움 → **알고리즘적 간소화** 기법.

### 절차 요약

1. 최소항을 **1의 개수** 기준으로 그룹화.
2. 1개만 다른 두 항을 합쳐 **Prime Implicant(소수 함의항)** 생성, 커버된 항에 체크.
3. 더 이상 합칠 수 없을 때까지 반복.
4. **PI 차트(Prime Implicant Chart)** 작성 → 필수 PI(Essential PI) 선정.
5. 모든 최소항을 커버하는 최소 집합 선택.

| 장점 | 단점 |
|------|------|
| 변수 수 제한 없음(소프트웨어 구현) | 수동 처리 복잡 |
| 결과가 수학적으로 최소 보장 | 지수적 계산 복잡도 |

## 6. 계산 예제 (3변수 KMap 완전 풀이)

F(A,B,C) = Σm(0,2,4,5,6)

```
      BC
AB  | 00 | 01 | 11 | 10 |
 0  |  1 |  0 |  0 |  1 |   ← m0(000), m2(010)
 1  |  1 |  1 |  0 |  1 |   ← m4(100), m5(101), m6(110)

그루핑:
① {m0, m2, m4, m6} → B=0, CD-any → C=0 (모두 C=0)
  → C' (4셀: A-don't, B-don't, C=0)

② {m4, m5} → A=1, B=0, C-don't → AB'

최종: F = C' + AB'

검증:
  C=0 → 항상 F=1 ✓
  A=1,B=0,C=1 (m5) → AB'=1 ✓
  m1,m3,m7: C=1이고 AB'≠1이면 F=0 ✓
```

## 7. 실무 연결과 주의점

- **FPGA LUT 절약**: 4입력 LUT 1개 = 진리표 16엔트리. 간소화된 SOP는 LUT 체이닝을 줄여 **전파 지연** 감소 및 **타이밍 클로저** 달성에 직결.
- **AURIX TC365 상태 디코더**: 다중 운전 모드(G2V/V2G/V2L) 전환 조건 식을 부울 논리로 작성할 때, don't-care 미사용 시 불필요한 게이트 증가 → MCU 코드에서도 if-else 분기 수 증가.
- **AUTOSAR 모드 관리(BswM)**: 복수 신호 조합으로 모드 전환 규칙 작성 → 진리표+KMap으로 규칙 최소화 후 룩업 테이블 형태로 구현 권장.
- **Don't care 오용 주의**: HW 인터페이스에서 don't-care로 처리한 입력이 실제 발생하면 정의되지 않은 출력 → 안전 분석(FMEA) 시 don't-care 입력 가능성 검토 필수.

## 8. 자가 점검

1. F(A,B,C) = Σm(1,3,5,7)을 카르노맵으로 간소화하라.
2. 4변수 함수에서 8셀 그룹이 만들어지면 결과 항의 리터럴 수는?
3. don't-care 조건을 SOP 간소화에 활용하는 방법을 설명하라.
4. Quine-McCluskey에서 Essential Prime Implicant란 무엇인가?
5. POS 표준형에서 Y=0이 되는 행을 사용하는 이유를 설명하라.

<details><summary>정답</summary>

1. KMap 4셀 모두 1 → {m1,m3,m5,m7}: A-don't, C=1 → F = C
2. 8셀 그룹 → 없애지는 변수 수 3 → 리터럴 4-3 = 1개 (단일 리터럴 항)
3. KMap에서 X를 1로 취급해 더 큰 그룹(2^n 배수)을 만들어 더 많은 변수를 제거. X를 반드시 어떤 그룹에 포함시킬 필요는 없음.
4. 하나의 최소항을 오직 자신만 커버하는 소수 함의항. 반드시 최종 해에 포함해야 함.
5. 최대항 M_i는 해당 입력 조합에서 Y=0이 되는 인수. 이들을 AND로 묶으면 Y=0인 조건을 정확히 표현하는 POS형이 완성됨.
