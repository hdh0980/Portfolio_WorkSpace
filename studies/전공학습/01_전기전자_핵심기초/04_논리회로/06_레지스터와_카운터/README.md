# 레지스터와 카운터

## 학습 목표
- SISO·SIPO·PISO·PIPO 시프트 레지스터의 구조와 동작을 비교할 수 있다.
- 직렬 로드와 병렬 로드의 차이를 설명하고, 데이터 흐름을 타이밍 다이어그램으로 그릴 수 있다.
- 리플 카운터와 동기 카운터의 구조적 차이와 타이밍 특성을 설명할 수 있다.
- Modulo-N 카운터를 설계할 수 있다.
- 레지스터와 카운터가 MCU 주변장치(타이머, SPI, UART)에서 어떻게 활용되는지 설명할 수 있다.

---

## 1. 레지스터 (Register)

여러 개의 D-FF를 병렬 배치해 n비트 데이터를 한 번에 저장.

```
4비트 병렬 로드 레지스터:
CLK 엣지에서 {I3,I2,I1,I0} → {Q3,Q2,Q1,Q0} 동시 전송
LOAD=1: 입력 데이터 로드
LOAD=0: 현재 상태 보류
```

---

## 2. 시프트 레지스터 (Shift Register)

FF를 직렬 연결: 각 FF의 Q가 다음 FF의 D로 연결.

### 2.1 분류

| 종류 | 입력 | 출력 | 설명 |
|------|------|------|------|
| SISO | 직렬 | 직렬 | 기본 시프트. n클록 후 데이터 출력 |
| SIPO | 직렬 | 병렬 | n클록 직렬 수신 → 병렬 출력. SPI 수신 버퍼 |
| PISO | 병렬 | 직렬 | 병렬 로드 후 직렬 송출. SPI 송신 버퍼 |
| PIPO | 병렬 | 병렬 | 파이프라인 레지스터. 클록 정렬 지연 |

### 2.2 4비트 SIPO 타이밍 예

```
입력 직렬: 1 0 1 1 (MSB first)
CLK  : ↑  ↑  ↑  ↑
Q3Q2Q1Q0:
  초기 = 0000
  CLK1: 1000   (입력=1 → Q3으로 shift)
  CLK2: 0100   (입력=0 → shift right)
  CLK3: 1010   (입력=1)
  CLK4: 1101   (입력=1)
최종 Q3Q2Q1Q0 = 1011  → 병렬 출력
```

### 2.3 방향 제어 (Universal Shift Register)

MODE 입력으로 Left Shift / Right Shift / Parallel Load / Hold 중 선택.

---

## 3. 리플 카운터 (Ripple Counter)

T-FF(T=1 고정) 를 직렬 연결. 하위 비트 FF의 Q가 상위 비트 FF의 CLK로 입력.

```
FF0 CLK = 외부 클록
FF1 CLK = FF0의 Q (Q0 하강 엣지에 트리거)
FF2 CLK = FF1의 Q
...

4비트 리플 카운터 동작: 0000 → 0001 → 0010 → ... → 1111 → 0000
```

> 참고 도식: Mano & Ciletti, *Digital Design* 6판, Fig. 6.3.

#### 리플 카운터 타이밍 문제

- 각 FF는 이전 FF가 전환된 후에만 변경 → **누적 전파 지연**
- n비트 카운터 최악 지연 = n × t_FF
- 고속 동작 시 중간 글리치 상태 존재 (예: 0111→1000 전환 중 잠시 1001이 나타남)

---

## 4. 동기 카운터 (Synchronous Counter)

모든 FF가 **공통 CLK**에 동시에 트리거. 전파 지연 누적 없음.

### 4비트 동기 이진 카운터 Enable 방정식

```
T0 = 1              (Q0는 항상 Toggle)
T1 = Q0             (Q1은 Q0=1일 때 Toggle)
T2 = Q1·Q0          (Q2는 Q1=Q0=1일 때)
T3 = Q2·Q1·Q0       (Q3은 Q2=Q1=Q0=1일 때)
Carry_out = Q3·Q2·Q1·Q0  (캐스케이드용)
```

| 장점 | 단점 |
|------|------|
| 동시 전이, 글리치 없음 | 조합 논리 증가(AND 체인) |
| 고속 동작 가능 | 회로 면적 증가 |

---

## 5. Modulo-N 카운터

0에서 N-1까지 계수하고 N에서 0으로 리셋.

### 설계 방법 1: 리셋 방식 (비동기 클리어)

```
예) Mod-6 카운터 (0~5):
4비트 카운터 + NAND 게이트 감지: Q2·Q1 = 1 (6 = 0110)
이 조건 발생 시 CLR(비동기 리셋) 활성 → 0000으로 복귀
단점: 6이 극히 짧게 나타났다 사라지는 글리치
```

### 설계 방법 2: 상태 할당 (동기 로드)

```
카운터가 N-1 상태에서 다음 클록에 0을 로드(LOAD=1).
글리치 없음. 선호 방법.
```

### 주요 Modulo-N 응용

| N | 용도 |
|---|------|
| 60 | 초·분 카운터 |
| 24 | 시간 카운터 |
| 10 | BCD 카운터 |
| 12 | 시간 (12시간제) |
| 2^n | PWM 분해능 레지스터 |

## 6. 계산 예제

**예제 1.** 3비트 리플 카운터에서 000→111 전환까지 걸리는 클록 사이클과 최악 전파 지연.

```
3비트 카운터 = 8개 상태(0~7)
000→001: 1 CLK
...
111→000(overflow): 1 CLK
최악 지연: 111→000 전환 시 FF0 → FF1 → FF2 순 전파
최악 전파 지연 = 3 × t_FF  (단, t_FF는 각 FF의 전파 지연)
```

**예제 2.** Mod-10 BCD 카운터를 4비트 동기 카운터 + 동기 로드로 구현.

```
상태 1001(=9)에서:
  감지 조건: Q3·Q0 = 1 (9 = 1001, Q3=1, Q0=1)
  다음 CLK에서 LOAD=1, 데이터 입력 = 0000
→ 0000으로 동기 복귀
중간 글리치 없음 ✓
```

## 7. 실무 연결과 주의점

- **AURIX TC365 GTM(Generic Timer Module)**: 고해상도 타이머 = 내부 카운터 레지스터. PWM 생성 시 Mod-N 카운터로 주기·듀티 제어. 62.5kHz PWM = 62.5MHz CLK / 1000카운터.
- **SPI 시프트 레지스터**: TC365 QSPI 모듈이 PISO(TX) + SIPO(RX) 시프트 레지스터 구조. 8/16/32비트 프레임 전송 = 8/16/32 클록 사이클.
- **UART 보레이트**: 비동기 직렬 통신 = SISO 시프트 레지스터 + 보레이트 클록(Mod-N 분주기).
- **OBC 인터리브드 PFC 위상 제어**: 2채널 스위칭 신호는 동기 카운터 기반 PWM의 위상 오프셋으로 생성(180° = 카운터 주기/2 오프셋).
- **리플 카운터 금지**: 자동차 안전 시스템(ISO26262 ASIL B 이상)에서 리플 카운터는 글리치로 인한 비결정적 출력 → 반드시 동기 카운터 사용 또는 동기화 후 사용.

## 8. 자가 점검

1. SIPO와 PISO 레지스터의 사용 예를 하나씩 들어라.
2. 4비트 리플 카운터에서 0111→1000 전환 중 발생할 수 있는 글리치 상태를 설명하라.
3. 동기 카운터가 리플 카운터보다 고속에 유리한 이유는?
4. Mod-12 카운터를 동기 로드 방식으로 설계할 때, 리셋 감지 조건(4비트 기준)을 쓰라.
5. 시프트 레지스터로 직렬-병렬 변환 시 n비트를 출력하려면 최소 몇 클록이 필요한가?

<details><summary>정답</summary>

1. SIPO: SPI 슬레이브 수신 버퍼 (8클록 동안 비트 수신 → 8비트 병렬 레지스터 출력). PISO: SPI 마스터 송신 버퍼 (8비트 병렬 로드 → 1비트씩 직렬 출력).
2. 0111(7)→1000(8) 전환 시 FF0가 먼저 0→1을 거쳐야 FF1이 반응. 리플 순서에 따라 순간적으로 0110, 0100, 0000 같은 중간 상태가 수 ns 동안 나타날 수 있음.
3. 동기 카운터는 모든 FF가 동일 CLK에 동시 트리거 → 최악 지연 = 조합 논리(AND 체인) 1단계. 리플은 n단 직렬 지연 누적.
4. 1100(=12) 감지: Q3·Q2 = 1, 이 순간 LOAD=1로 0000 로드. (12 = 1100이므로 Q3=1, Q2=1, Q1=0, Q0=0)
5. n클록. 첫 클록에 MSB가 Q[n-1]에 저장되고 n번째 클록에 LSB가 Q[0]에 저장됨.
