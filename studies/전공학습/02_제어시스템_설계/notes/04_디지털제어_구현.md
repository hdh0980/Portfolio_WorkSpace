# 04. 디지털 제어 구현

## 1. 연속시간 설계에서 코드까지

디지털 제어기는 연속 신호를 샘플링하고, 제한된 계산시간 안에 제어식을 실행한
후 PWM 또는 DAC로 조작량을 갱신한다. 따라서 연속시간 전달함수만 맞아도 실제
제어가 자동으로 같아지는 것은 아니다.

```text
센서 샘플링 → 보정/필터 → 오차 계산 → 제어기 → 제한/보호 → PWM 갱신
```

## 2. 샘플링과 aliasing

샘플링 주파수가 $f_s$이면 Nyquist 주파수는 $f_s/2$이다. 그보다 높은 아날로그
성분은 낮은 주파수로 접혀 보일 수 있으므로 ADC 앞단의 anti-aliasing 필터와
제어 대역을 함께 정한다.

샘플링 주기는 목표 폐루프 대역폭보다 충분히 빨라야 한다. `10~20배`는 초기
검토용 경험칙일 뿐이며, 실제 값은 계산 지연, PWM 동기화, 센서 필터와 함께
주파수응답 또는 이산시간 모델로 검증한다.

## 3. ZOH와 이산화

Zero-order hold는 다음 갱신 시점까지 제어 출력을 유지한다. 연속 제어기를
이산화할 때 대표적으로 다음 방법을 비교한다.

- Forward Euler: 단순하지만 안정영역이 제한적
- Backward Euler: 수치적으로 보수적
- Tustin(bilinear): 주파수 축 왜곡이 있지만 안정 특성을 보존하기 쉬움
- ZOH 기반 plant 이산화: 샘플·홀드의 영향을 플랜트 모델에 반영

방법 이름만 기록하지 말고 샘플링 시간과 이산화 방식, prewarping 여부를 모델
메타데이터로 남긴다.

## 4. 위치형·증분형 PI

단순한 Forward Euler 적분을 사용하는 위치형 PI는 다음과 같이 쓸 수 있다.

$$
x_I[k]=x_I[k-1]+K_iT_s e[k]
$$

$$
u_{raw}[k]=K_p e[k]+x_I[k]
$$

같은 가정에서 증분형은 다음과 같다.

$$
u[k]=u[k-1]+K_p(e[k]-e[k-1])+K_iT_s e[k]
$$

실제 코드에는 출력 제한, anti-windup, enable 조건, 초기화, 유효하지 않은 센서
값 처리, 부호와 단위를 추가해야 한다.

## 5. ISR 실행 순서 예시

```c
void ControlISR(void)
{
    measurement = ReadAndScaleAdc();
    filtered = UpdateMeasurementFilter(measurement);
    error = reference - filtered;

    p_term = kp * error;
    i_state += ki_ts * error;
    raw_output = p_term + i_state + feedforward;

    limited_output = Clamp(raw_output, output_min, output_max);
    i_state += kaw * (limited_output - raw_output);

    if (!control_enabled || sensor_invalid) {
        limited_output = safe_output;
        i_state = tracked_initial_value;
    }

    UpdatePwm(limited_output);
}
```

이 코드는 구조 예시다. 실제 구현에서는 고정소수점 범위, overflow, 실행시간,
re-entrancy, PWM update 시점과 fault 우선순위를 프로젝트 규칙에 맞춰 정의한다.

## 6. 구현 검증

- 연속·이산 모델 응답 비교
- 샘플링 주기와 1~2 tick 지연 민감도
- ADC 양자화·잡음·센서 offset
- 출력 포화와 anti-windup 회복
- 연산 overflow 및 NaN/Inf 방어
- 실행시간 worst case와 deadline
- enable/disable 및 모드 전환 시 bumpless transfer
