# 06. 고급제어 확장

PID 이후의 기법은 더 복잡해서 쓰는 것이 아니라, PID만으로 분리하기 어려운
문제를 해결하기 위해 선택한다. 먼저 해결하려는 문제와 추가되는 모델 의존성을
명확히 한다.

## 1. Feedforward control

측정 가능한 외란 또는 목표 궤적을 이용해 필요한 제어량을 미리 계산한다.
피드백은 모델 오차를 보정하고, feedforward는 예상 가능한 변화에 빠르게 반응한다.

예시:

- 입력전압 변화에 따른 듀티 보상
- 부하 전력 지령에 따른 전류 지령 생성
- grid voltage를 이용한 전류 제어 feedforward

Feedforward만으로는 모델 오차를 없앨 수 없으므로 피드백과 함께 검증한다.

## 2. Disturbance Observer(DOB)

명목 플랜트와 실제 출력의 차이를 이용해 등가 외란을 추정하고 보상한다. 장점은
외란 억제 구조를 명시적으로 만들 수 있다는 점이고, 위험은 모델 역함수와 필터가
고주파 잡음이나 비최소위상 특성과 충돌할 수 있다는 점이다.

확인 항목:

- 명목 모델의 유효 동작범위
- Q-filter의 대역폭
- 센서 잡음과 계산 지연
- 추정 외란 포화와 진단

## 3. Repetitive control

주기가 알려진 오차를 반복적으로 학습하여 동일 주파수와 고조파의 정상상태 오차를
줄인다. 계통 연계 인버터처럼 주기적 지령·외란이 있는 시스템에서 후보가 될 수
있다. 한 주기 지연 메모리, 안정화 필터, 계통 주파수 편차에 대한 민감도를 본다.

## 4. System identification

입출력 데이터로 모델 파라미터 또는 전달함수를 추정한다. 최소제곱법을 사용할 때도
데이터의 excitation, 센서 offset, 동작점, 포화 구간을 구분해야 한다.

```text
시험 설계 → 데이터 정제 → 모델 구조 선택 → 파라미터 추정
→ 별도 데이터로 검증 → 유효 범위 기록
```

훈련 데이터에서만 오차가 작으면 제어 설계용 모델로 충분하지 않다.

## 5. Adaptive control

플랜트 파라미터 변화에 맞춰 제어 파라미터를 조정한다. 적응 속도보다 먼저 다음을
정한다.

- 추정 가능한 파라미터와 물리 범위
- excitation이 부족할 때 update 정지 조건
- projection, dead zone 등 파라미터 발산 방지
- 적응 제어 실패 시 안전한 고정 gain fallback

## 6. 선택 가이드

| 문제 | 우선 검토할 방법 |
|---|---|
| 정상상태 오차 | PI 또는 적분 상태 추가 |
| 측정 가능한 외란의 빠른 영향 | Feedforward |
| 미측정 저주파 외란 | DOB 또는 observer augmentation |
| 주기적 오차·고조파 | Repetitive/PR control |
| 플랜트 파라미터 불명확 | System identification |
| 파라미터가 운전 중 크게 변화 | Gain scheduling 또는 adaptive control |
| 다변수 상태·성능 최적화 | State feedback, LQR/LQG |
